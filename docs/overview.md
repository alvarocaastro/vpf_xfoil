# Project Overview

## Functional Explanation

The project evaluates a variable-pitch fan (VPF) concept through a staged aerodynamic and propulsion workflow. The main engineering idea is that changing blade pitch can keep each blade section closer to its optimal angle of attack across takeoff, climb, cruise, and descent, improving aerodynamic efficiency relative to a fixed-pitch fan.

This is not a web application or packaged library. It is a technical analysis pipeline driven by Python scripts and configuration files. Outputs are written to disk for inspection, plotting, reporting, and downstream stages.

## End-to-End Behavior

The system:

1. Reads aerodynamic and engine parameters from YAML files.
2. Loads airfoil geometries from `.dat` files.
3. Runs XFOIL to evaluate candidate NACA airfoils.
4. Selects the best candidate using a weighted score.
5. Runs final polars for four flight phases and three blade sections.
6. Corrects raw XFOIL polars for compressibility effects.
7. Computes aerodynamic metrics and fixed-pitch penalties.
8. Applies cascade and 3D rotational corrections.
9. Computes blade twist, pitch adjustment, off-design incidence, and stage loading.
10. Estimates reverse-thrust mechanism weight and cruise SFC penalty.
11. Estimates SFC reduction and mission fuel burn impact.
12. Writes tables, figures, and summaries to `results/`.

## Expected Inputs

| Input | Path | Purpose |
|---|---|---|
| Aerodynamic configuration | `config/analysis_config.yaml` | Flight conditions, Mach, Reynolds, Ncrit, fan geometry, blade geometry, XFOIL settings. |
| Engine and mission configuration | `config/engine_parameters.yaml` | Baseline SFC, fan efficiency, BPR, mission phases, reverse-thrust assumptions. |
| Airfoil catalog | `config/airfoils.yaml` | Candidate names, `.dat` files, families, comments. |
| Airfoil coordinate files | `data/airfoils/*.dat` | Geometry files consumed by XFOIL. |
| XFOIL executable | External binary | Generates viscous 2D airfoil polars. |

## Main Processing Stages

| Result Stage | Description |
|---|---|
| Stage 1 | Airfoil selection using mission-weighted XFOIL results. |
| Stage 2 | Final XFOIL simulations and pitch-map generation. |
| Stage 3 | Compressibility correction of polars. |
| Stage 4 | Aerodynamic performance metrics and figures. |
| Stage 5 | Pitch kinematics, cascade, rotational effects, twist, and loading. |
| Stage 6 | Reverse-thrust mechanism weight and SFC impact. |
| Stage 7 | SFC analysis, mission fuel burn, and GE9X parametric validation. |
| Sensitivity | Additional sweep over `tau` and RPM deviation. |

`run_analysis.py` includes an initial cleanup step, so its CLI stages are numbered 1 to 8 while result folders are numbered 1 to 7.

## Generated Outputs

The main output tree is:

```text
results/
  stage1_airfoil_selection/
  stage2_xfoil_simulations/
  stage3_compressibility_correction/
  stage4_performance_metrics/
  stage5_pitch_kinematics/
  stage6_reverse_thrust/
  stage7_sfc_analysis/
  sensitivity/
```

Outputs include:

- Raw and corrected aerodynamic polars.
- CSV tables for metrics, kinematics, stage loading, SFC, mission fuel burn, and sensitivity.
- PNG figures for diagnostics, comparisons, validation, and reporting.
- Text summaries for pipeline stages.
- One LaTeX table generated by Stage 7.

## Main Use Cases

- Compare NACA airfoil candidates for a fan-blade application.
- Quantify fixed-pitch aerodynamic penalty relative to VPF.
- Study sensitivity to efficiency-transfer assumptions and RPM offsets.
- Generate technical figures and tables for a thesis or engineering report.
- Extend the analysis with new airfoils, flight conditions, or higher-fidelity reverse-thrust modeling.

## Detected Assumptions

| Assumption | Evidence |
|---|---|
| The reference engine is GE9X-class with a 3.40 m fan and 16 blades. | Present in README and YAML comments. |
| The flight mission is represented by four discrete operating conditions. | `analysis_config.yaml`. |
| Stage communication occurs through files in `results/`. | `pipeline/contracts.py` and `run_analysis.py`. |
| XFOIL runs at a reference Mach and Stage 3 applies target Mach corrections. | `reference_mach` and `target_mach`. |
| 2D aerodynamic gains are damped before SFC conversion. | `profile_efficiency_transfer` and SFC caps. |
| Reverse-thrust aerodynamic feasibility is not fully validated in the current main Stage 6 workflow. | `run_reverse_thrust.py` summary text. |

Items that require external validation are marked as **pending confirmation** in the relevant documents.
